## 2. 다음 프로젝트 개요

### 프로젝트명
**경제 시황 텔레그램 브리핑 봇**

### 핵심 개념
> 내가 관심사를 사전에 정의해두면, 시스템이 알아서 수집·분석해서 하루 3번 텔레그램으로 전송

퍼플렉시티와 차이:
- 퍼플렉시티: 내가 물어봐야 답함 (Reactive)
- 이 봇: 알아서 챙겨서 보내줌 (Proactive)

---

## 3. 하루 3회 브리핑 구성

### ① 오전 7시 — 해외 시장 브리핑 + 국내 장 전망
```
[전일 해외 시장 현황]
- 미국/유럽/아시아 주요 지수 등락 및 원인
- 주요 경제 지표 발표 내용
- 핵심 이슈 요약

[오늘 국내 시장 진단]
- 해외 흐름이 국내 시장에 미칠 영향
- 오늘 주목해야 할 섹터/종목
- 국내 주요 일정 (실적 발표, 경제 지표 등)
```

### ② 오전 12시 — 오전장 결산 + 오후장 전망
```
[오전장 주요 이벤트]
- 오전장에서 발생한 주요 등락 및 원인
- 특징적인 섹터/종목 움직임
- 외국인/기관 매매 동향

[오후장 및 익일 전망]
- 오후장 방향성 예측
- 내일 시장에 영향 줄 변수
```

### ③ 오후 6시 — 당일 결산 + 익일 영향 분석
```
[금일 국내 시장 결산]
- 주요 지수 최종 등락 및 원인
- 핵심 이벤트 정리
- 섹터별 흐름 요약

[중기 관점 분석]
- 현재 이슈들이 중기적으로 어떻게 흘러갈 것인가
- 내일 시장에 미칠 영향
- 주목해야 할 변수
```

## 4. 기술 스택

```
[데이터 수집]
- The Guardian API     → 해외 경제 뉴스
- BigKinds API         → 국내 경제 뉴스
- (추가 검토) 금융 데이터 API → 실제 지수/가격 데이터

[AI 분석]
- Gemini API (무료 티어) → 뉴스 요약 + 시황 분석

[전송]
- Telegram Bot API → 하루 3회 메시지 전송

[인프라]
- Python 백엔드
- Contabo 리눅스 서버
- Cron 스케줄링 (7시 / 12시 / 18시)
```

---

## 5. 향후 고도화 방향

- 관심 섹터/종목 사전 정의 → 맞춤형 브리핑
- 임베딩 기반 벡터 검색 → 진짜 맥락 기반 뉴스 필터링
- 주요 이슈 연속성 추적 → "어제 언급된 이슈가 오늘 어떻게 됐는지"
- 메시지 포맷 커스터마이징 → 상세 형식은 프로젝트 진행 중 결정
